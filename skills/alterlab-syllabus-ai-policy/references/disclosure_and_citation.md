# Disclosure & Citation Templates for Generative-AI Use

Verbatim, source-backed citation templates plus a student AI-use declaration
block. A Restricted or Permitted tier (see `tier_framework.md`) is only complete
when it tells students **exactly** how to disclose and cite — never "cite it
appropriately."

## 1. APA Style

APA credits the **maker** of the model as author, not the tool, and not the
student. Reference-list template (verbatim from APA Style, *How to cite
ChatGPT*):

```
OpenAI. (2023). ChatGPT (Mar 14 version) [Large language model]. https://chat.openai.com/chat
```

- **Author:** the company that made the model (e.g. `OpenAI`).
- **Date:** the year of the version used.
- **Title:** the tool name in italics, with the version in parentheses.
- **Bracketed description:** `[Large language model]`.
- **Source:** the URL of the tool.
- **In-text:** parenthetical `(OpenAI, 2023)`; narrative `OpenAI (2023)`.

APA additionally advises documenting the **exact prompt and the generated text**
in the paper (Methods section or an appendix). Adapt the maker, version, and URL
to the tool the student actually used; do not invent a version string or URL.

> Source: APA Style, *How to cite ChatGPT*, apastyle.apa.org.

## 2. MLA Style

MLA uses its template of core elements and treats the AI tool as the **container,
not the author** (MLA explicitly does not recommend listing the tool as author).
Works Cited template (per MLA Style Center):

```
"<prompt text>" prompt. ChatGPT, <version e.g. 13 Feb. version>, OpenAI, <date>, <general URL>.
```

Core elements MLA names:

1. **Title of source** — describe what the AI generated (include the prompt).
2. **Title of container** — the AI tool, italicized (e.g. *ChatGPT*).
3. **Version** — the version and its date.
4. **Publisher** — the company that made the tool.
5. **Date** — when the content was generated.
6. **Location** — the general URL.

MLA also requires writers to **acknowledge functional uses** of AI (editing
prose, translating) in a note or in the text, even when nothing is quoted.

> Source: MLA Style Center, *How do I cite generative AI in MLA style?*,
> style.mla.org.

## 3. Chicago (note option)

For courses using Chicago notes, disclose AI use in a note. Chicago's published
guidance treats AI-generated text like personal communication and recommends a
**note** rather than a bibliography entry, e.g.:

```
1. Text generated by ChatGPT, OpenAI, <date>, <URL>.
```

> Confirm the current wording against the Chicago Manual of Style Online Q&A
> ("Citation, Documentation of Sources") before publishing — this option is
> offered as a starting note form, not a fixed template. If you cannot confirm
> it, fall back to APA or MLA above.

## 4. The student AI-use declaration block

Append this to each Restricted/Permitted submission. It is the practical artifact
that makes disclosure enforceable. Fill-ins in `<…>`:

```
AI-Use Declaration
- Did you use any generative-AI tool on this submission?  [ ] No   [ ] Yes
- If yes, which tool(s) and version(s):  <e.g. ChatGPT, GPT-4o, May 2025>
- For which parts / sub-tasks:  <e.g. brainstorming the outline only>
- How you verified anything the tool produced:  <e.g. checked every cited source against the library catalog>
- Citation(s) included per the course documentation standard:  [ ] Yes  [ ] N/A
I confirm the assessed work is my own and that any AI use complied with the
tier stated for this assignment in the syllabus.
Signature: ____________________   Date: ________
```

## 5. What disclosure must always state

Whatever the format, a complete disclosure names:

- **Which tool and version** was used,
- **For what** (which sub-tasks),
- **How the student verified** the output (AI can fabricate facts and citations),
- and includes the **citation** in the course's documentation standard.

A policy that says "disclose AI use" without specifying these four is flagged by
`scripts/policy_lint.py` as an incomplete disclosure clause.
